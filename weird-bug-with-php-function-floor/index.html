Weird bug with PHP function floor()

In my project, I have to deal with a small problem:
Input: Some strings/numbers with value $"a_1a_2..a_n.xy"$ ($a_1, a_2, a_n, x, y$ are numbers in [0..9])

For examples, "12345.04", "384.13", etc.
Output: Two numbers that were substrings from input splited by "." (12345 and 04, 384 and 13, etc.)

OK, that's quite easy. At the beginning, I came up with this solution:

$a = '1704.09';
$b = floor($a);    // first number
$c = (int)($a * 100) - $b * 100;    // second number
echo $a."<br />".$b."<br />".$c;

Everything seems to be fine, function floor() giving you the lowest bounding integer of a float. Lately, I realized that my code provides wrong results with some inputs ended by ".01": "77655.01", "17064.01", etc. Of course, my code has to be changed:

$a = '1704.01';
$array_a = explode(".", $a);
$b = (int) $array_a[0];    // first number
$c = (int) $array_a[1];    // second number
echo $a."<br />".$b."<br />".$c;

The following code shows that from $0$ to $10^7$, there are exactly 610143 numbers $x$ have property: $100 * (x + 0.01) \neq \ "x01"$

$counter = 0;
for($i = 0; $i < 10000000; $i++) {  
    $a = $i.'.'.'01';
    $f = floor($a);
    $b = (int)($a * 100) - $f * 100;
    if($b == 0) {
        echo $i;
        echo "<br />";
        $counter++;
    }
}
echo "=== DONE ===<br />";  
echo $counter;

Interesting, so be careful with PHP floor() function if you use it :)
